# Responses API Migration Plan for Presentation AI

## Overview
Migrate the ALLWEONE Presentation AI application from Chat Completions API to OpenAI's newer Responses API to unlock enhanced features and better performance.

## Current State Analysis
The application currently uses OpenAI SDK with Chat Completions for:
- AI-powered content generation
- Image generation (DALL-E)
- Text-to-diagram generation
- Presentation outline creation

## Features to Add/Enhance with Responses API

### 1. Native Web Search Tool
**Current**: Uses Tavily API (optional, separate service)
**Enhancement**: Migrate to OpenAI's built-in `web_search` tool
- Remove dependency on Tavily API
- Better integration with content generation
- Real-time search results within AI responses
- Cost savings (no separate API key needed)

**Files affected**:
- `src/ai/tools/search.ts`

### 2. Integrated Image Generation
**Current**: Separate DALL-E API calls
**Enhancement**: Use `image_generation` tool within responses
- Generate images inline during content creation
- Better context-aware image generation
- Single API call for both text and images

**Files affected**:
- `src/app/api/presentation/generate-image-slides/`
- `src/components/presentation/editor/`

### 3. Enhanced Multi-turn Outline Generation
**Current**: Separate API calls for outline and content
**Enhancement**: Use `previous_response_id` for conversation chaining
- Maintain context between outline and slide generation
- Better consistency across presentation
- Simpler state management

**Files affected**:
- `src/app/api/presentation/outline/`
- `src/app/api/presentation/generate/`

### 4. Code Interpreter for Chart Generation
**Current**: Basic chart components
**Enhancement**: Use `code_interpreter` tool to generate charts/diagrams
- Dynamic chart generation from data
- Python/R code execution for complex visualizations
- Export charts to presentation slides

**Files affected**:
- `src/ai/tools/presentation/tools.ts`
- `src/components/presentation/slides/`

### 5. Reasoning Models Integration
**Current**: Uses GPT-4/4o models
**Enhancement**: Use GPT-5 with improved reasoning
- Higher quality content generation
- Better structured output
- 3% improvement in accuracy (per OpenAI docs)

**Files affected**:
- Model selection throughout the app
- `src/ai/lib/model-picker.ts`

### 6. Stateful Conversation Memory
**Current**: Manual session management in database
**Enhancement**: Use `store: true` for native state management
- Reduced database operations
- Better conversation continuity
- Automatic context preservation

**Files affected**:
- `src/app/_actions/notebook/presentation/`
- Session management code

### 7. Streaming Response Updates
**Current**: Non-streaming responses with progress indicators
**Enhancement**: Use Responses API streaming for real-time generation
- Users see tokens appear as they're generated
- Live slide content building
- Progress streaming for better UX

**Files affected**:
- `src/app/api/presentation/generate/route.ts`
- `src/app/api/presentation/outline/route.ts`
- `src/states/presentation-state.ts`
- Frontend components for streaming display

### 8. Parallel Tool Execution
**Current**: Sequential API calls for different features
**Enhancement**: Use Responses built-in parallel tool execution
- Web search + image generation in single request
- Faster overall generation time
- Coordinated outputs

**Files affected**:
- All presentation generation API endpoints

## Implementation Steps

### Phase 1: Core Migration
1. Update `src/env.js` to add `OPENAI_RESPONSES_API_KEY` (same key works for both APIs)
2. Create `src/lib/openai/responses.ts` wrapper for Responses API
3. Update API routes to use Responses API format

### Phase 2: Web Search Integration
1. Replace Tavily search tool with native `web_search` tool
2. Update presentation generation to use integrated search
3. Remove Tavily API dependency (optional)

### Phase 3: Enhanced Features
1. Add inline image generation via `image_generation` tool
2. Implement code interpreter for charts
3. Add conversation state management with `store: true`

### Phase 4: Model Updates
1. Add GPT-5 model support
2. Update model selection UI
3. Configure reasoning effort levels

### Phase 5: Streaming & Parallel Tools
1. Implement streaming response handler
2. Add SSE (Server-Sent Events) endpoint for live updates
3. Enable parallel tool execution in presentation generation
4. Frontend streaming display components

## Streaming Implementation Details

### Backend Streaming Endpoint
Create `src/lib/openai/stream-handler.ts`:
- Transform Responses API stream to SSE format
- Send incremental updates for slide content
- Handle tool call streaming events

### Frontend Streaming Display
Update presentation state to:
- Receive stream events via EventSource
- Update slide content in real-time
- Show typing indicators for each section

### Files for Streaming
- `src/app/api/presentation/stream/route.ts` (new)
- `src/hooks/presentation/useStreamPresentation.ts` (new)
- `src/states/presentation-state.ts` (update)

## Benefits
- **Lower costs**: 40-80% better cache utilization
- **Better performance**: Improved reasoning models
- **Simplified architecture**: Fewer external dependencies
- **Native tools**: No need for custom function implementations
- **Future-proof**: Ready for upcoming OpenAI features

## Risks & Considerations
- Breaking changes in function/tool format
- Need to update structured output handling (`text.format` vs `response_format`)
- Backward compatibility for existing presentations
- Testing required for all API endpoints
